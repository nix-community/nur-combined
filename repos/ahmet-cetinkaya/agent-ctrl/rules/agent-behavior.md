## Code Editing Rules

### No Change Tracking Comments

- **Never add comment lines that only indicate changes:**
  - ❌ `// Added`
  - ❌ `// Changed`
  - ❌ `// Fixed`
  - ❌ `// Updated`
  - ❌ `// Removed`
  - ❌ Any similar one-word change annotations

- Change history belongs in Git commit history, not in source code files.
- Only keep comments that explain _why_ code exists, not _when_ or _how_ it was changed.
